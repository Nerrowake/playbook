# Design System

Nerrowake design should feel clear, practical, and trustworthy. Interfaces should
serve the workflow before they express a brand.

## Product Feel

Nerrowake products should be:

- fast
- readable
- calm
- precise
- useful
- accessible
- visually restrained

## Interface Principles

- prioritize hierarchy and spacing
- make primary actions obvious
- keep repeated patterns consistent
- use plain language
- design empty, loading, error, and success states
- avoid decoration that competes with the task
- make dense information scannable

## Components

Reusable components should have clear responsibilities and predictable behavior.
Document component variants when they become shared across a project.

## Visual Notes

UI pull requests should include screenshots, screen recordings, or written visual
notes. The reviewer should be able to understand what changed without running
the branch.

## Brand Restraint

Nerrowake can have personality without making tools harder to use. Product
interfaces should not sacrifice clarity for atmosphere.
