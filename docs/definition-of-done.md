# Definition of Done

A feature is not complete until:

- it works as intended
- it has appropriate tests
- it has documentation
- it handles errors gracefully
- it follows project conventions
- it has been reviewed
- README or docs are updated if behavior changed
- changelog is updated if needed
- UI changes include screenshots or visual notes
- security and privacy implications are considered

## Applying This Standard

The definition of done should scale with the risk and size of the change. A small
documentation fix does not need the same process as a user-facing feature, but it
should still be reviewed with care.

## Done Means Owned

Done does not mean "code exists." Done means the change can be used, understood,
tested, maintained, and supported.
