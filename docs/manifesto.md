# Manifesto

Software should be clear enough to understand, reliable enough to trust, and
pleasant enough to keep using.

Nerrowake is not trying to sound larger than it is. It is a small,
founder-led studio with a practical engineering bias. The work should be steady,
specific, and useful.

## We Prefer

- clear names over clever abstractions
- maintainable systems over novel architecture
- correctness before performance tuning
- useful documentation over hidden knowledge
- thoughtful design over decoration
- explicit tradeoffs over vague ambition
- long-term ownership over short-term polish

## We Avoid

- complexity that does not earn its place
- hype-driven roadmaps
- fragile demos that cannot become products
- undocumented behavior
- clever code that future maintainers have to decode
- interfaces that look finished but do not help users

## The Work

Build the right thing carefully. Explain what it does. Test what matters. Leave
the project easier to understand than it was before.
